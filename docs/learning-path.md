# 学习路线

## 第一阶段：能识别变化点

先完成 Strategy、Factory、Adapter。重点观察：哪些代码会因为一个新渠道、新算法、新接口而反复修改。

## 第二阶段：能保护业务流程

完成 Template Method、State、Command。重点观察：流程稳定但步骤变化、状态变化、操作历史这三类问题。

## 第三阶段：能控制协作边界

完成 Observer、Decorator、Facade、Singleton。重点观察：对象之间什么时候应该认识彼此，什么时候应该只依赖一个小入口。

## AI 辅助开发建议

让 AI 帮你写代码前，先描述变化点，而不是直接说“用某某模式”。例如：

```text
这段 C++ 代码现在通过 if/else 处理三种计费方式，马上会新增更多计费方式。
请先指出变化点和坏味道，再给出一个小步重构方案，并保持现有测试通过。
```

让 AI 帮你 review 时，重点问：

```text
这个设计模式是否过度设计？
新增一个业务变体时，需要改哪些已有类？
有没有隐藏的生命周期、所有权或线程安全问题？
```
